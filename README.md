# nginx-manager — простое управление сайтами на VPS

Интерактивный bash-скрипт для упрощённого создания, удаления, включения/выключения сайтов (виртуальных хостов) в nginx. Также позволяет установить популярные веб-пакеты (PHP, MySQL, PostgreSQL, SQLite3, Certbot). Скрипт сам устанавливает себя в `/usr/local/bin/nginx-manager` при первом запуске.

## Особенности

- ✅ Интерактивное меню с цветовым оформлением  
- ✅ Создание сайта за 30 секунд  
- ✅ Поддержка статических сайтов и проксирования на внутренний порт (Node.js, Python, Go и т.д.)  
- ✅ Автоматическое создание папки `/var/www/<имя>` и тестовой страницы  
- ✅ Генерация конфигурации nginx в `sites-available` и симлинк в `sites-enabled`  
- ✅ Список только своих сайтов (сохраняется в `/etc/nginx/.my_sites`)  
- ✅ Удаление только тех сайтов, которые были созданы через скрипт  
- ✅ Проверка синтаксиса nginx и автоматическая перезагрузка  
- ✅ Установка недостающих пакетов (с выбором)  
- ✅ Самоустановка в систему (доступен как `sudo nginx-manager`)

## Установка за 1 минуту

Скопируйте скрипт на VPS и запустите:

```bash
curl -fsSL https://raw.githubusercontent.com/hargluk/nginx-manager/refs/heads/main/nginx-manager-installer.sh | sudo bash
```
Либо
```bash   
wget -O nginx-manager-installer.sh https://raw.githubusercontent.com/hargluk/nginx-manager/refs/heads/main/nginx-manager-installer.sh && chmod +x /usr/local/bin/nginx-manager-installer.sh && sudo ./nginx-manager-installer.sh
```

При первом запуске скрипт предложит установить себя в `/usr/local/bin/nginx-manager`. Согласитесь — после этого управление сайтами будет доступно одной командой:

```bash
sudo nginx-manager
```

> Если вы откажетесь от установки, скрипт продолжит работу, но вы не сможете вызывать его из любого места.

## Требования

- Ubuntu / Debian (или другой Linux с apt и nginx)
- Права sudo (root)
- Интернет для установки пакетов

## Меню

После запуска откроется главное меню:

    ======================================
          Управление сайтами nginx
    ======================================
    1. Создать сайт
    2. Список моих сайтов
    3. Удалить сайт
    4. Вкл/Выкл сайт
    5. Перезагрузить nginx
    6. Установить пакеты (PHP, MySQL, PostgreSQL, SQLite3, Certbot)
    0. Выход
    ======================================

### 1. Создать сайт

- Введите короткое имя (например, `mysite`) — оно будет использовано для папки и имени конфига.
- Введите домен (например, `example.com`). Домен будет подставлен в `server_name`.
- Выберите тип:
  - **Статика** — обычный статический сайт (HTML/CSS/JS).
  - **Прокси** — перенаправление на локальный порт (например, для приложения на Node.js, порт 3000).
- При создании автоматически:
  - Создаётся папка `/var/www/<имя>`
  - Создаётся `index.html` (если пусто)
  - Генерируется конфиг nginx в `/etc/nginx/sites-available/<имя>.conf`
  - Создаётся симлинк в `sites-enabled`
  - Проверяется конфиг и перезагружается nginx

### 2. Список моих сайтов

Показывает все сайты, созданные через этот скрипт, их домены и статус (включён/выключен).

### 3. Удалить сайт

Удаляет только те сайты, которые были созданы через скрипт. При удалении спрашивает, нужно ли удалить папку `/var/www/<имя>`.

### 4. Вкл/Выкл сайт

Переключает симлинк в `sites-enabled` — сайт остаётся в `sites-available`, но перестаёт обрабатываться nginx (или снова включается).

### 5. Перезагрузить nginx

Проверяет конфигурацию `nginx -t` и перезагружает сервис.

### 6. Установить пакеты

Показывает список пакетов (PHP, MySQL, PostgreSQL, SQLite3) и позволяет выбрать, что установить. Можно установить всё недостающее или выборочно. Скрипт сам запустит сервисы.

### 0. Выход

Завершает работу.

## Структура, создаваемая скриптом

    /var/www/
      └── <имя>/
          └── index.html

    /etc/nginx/
      ├── sites-available/
      │   └── <имя>.conf
      ├── sites-enabled/
      │   └── <имя>.conf -> ../sites-available/<имя>.conf
      └── .my_sites            (файл со списком имён, созданных скриптом)

## Пример конфигурации для статики

    server {
        listen 80;
        listen [::]:80;
        server_name example.com www.example.com;
        root /var/www/mysite;
        index index.html index.htm index.php;
        location / {
            try_files $uri $uri/ =404;
        }
    }

## Пример конфигурации для прокси

    server {
        listen 80;
        listen [::]:80;
        server_name api.example.com www.api.example.com;
        location / {
            proxy_pass http://127.0.0.1:3000;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
    }

## Безопасность

- Все операции требуют `sudo`.
- Удаление разрешено только для сайтов, созданных этим скриптом (проверка по `/etc/nginx/.my_sites`).
- Скрипт не редактирует чужие конфигурации.

## Удаление скрипта

Если вы больше не хотите использовать `nginx-manager`, просто удалите файл:

```bash
sudo rm /usr/local/bin/nginx-manager
```

Созданные сайты и конфиги останутся нетронутыми.
